#### Group 1: Carriage Return `\r` Shielding Semicolons
- **Pattern:** Accidental double semicolon (`;;`) or missing statement terminator before keywords.
- **Affected Procedures:** `Board_DeleteDepartAllowAccess`, `Board_GetAllBoardContentsByBoardList`, `Board_GetBoardContents`, `Board_GetBoardContents_BK20181227`, `Board_InsertAndroidDevice`, `Board_InsertFile`, `Board_InsertIOSDevice`, `Board_InsertReplyFile`, `Board_Mobile_Search`, `Board_UpdateAllowAccess`, `Board_UpdateConfig`, `Board_UpdateLevelRand`, `Board_UpdateNoticePermission`, `Board_Web_Search`
- **Example MSSQL Syntax:** `DELETE FROM Table WHERE ID=@ID`
- **Expected PostgreSQL Syntax:** `DELETE FROM Table WHERE ID=ID;`
- **Recommended Fix Location:** In [Converter.cs](file:///e:/pg_converter_ui/Converter.cs#L964) line 964, normalize all line endings to `\n` using `.Replace("\r\n", "\n").Replace('\r', '\n')` before running regexes.

#### Group 2: Unconverted T-SQL Cursors
- **Pattern:** Mismatched or missing cursor syntax leaving bare `OPEN`, `FETCH`, `CLOSE`, `DEALLOCATE` statements in PL/pgSQL.
- **Affected Procedures:** `Board_DownBoard`, `Board_DownFolder`, `Board_UpBoard`, `Board_UpFolder`
- **Example MSSQL Syntax:** `DECLARE CursorName CURSOR FOR SELECT ...`
- **Expected PostgreSQL Syntax:** `FOR rec IN SELECT ... LOOP ... END LOOP;`
- **Recommended Fix Location:** Add cursor-to-loop translation logic in `ConvertControlFlow` or handle it in a pre-process regex inside `BodyConverter.cs`.

#### Group 3: Redundant ORDER BY inside UNION branches
- **Pattern:** `ORDER BY` statement directly before `UNION ALL` inside subqueries.
- **Affected Procedures:** `Board_DownBoardByUser`, `Board_DownFolderByUser`, `Board_UpBoardByUser`, `Board_UpFolderByUser`
- **Example MSSQL Syntax:** `SELECT * FROM A ORDER BY Col UNION ALL SELECT * FROM B ORDER BY Col`
- **Expected PostgreSQL Syntax:** `SELECT * FROM A UNION ALL SELECT * FROM B` (inner ORDER BY removed)
- **Recommended Fix Location:** Add a regex rule in `BodyConverter.cs` to match and strip `ORDER BY` clauses that immediately precede `UNION` or `UNION ALL` statements.

#### Group 4: SQL Server-specific XML Processing
- **Pattern:** `EXEC sp_xml_preparedocument` and `OPENXML` statements.
- **Affected Procedures:** `Board_InsertNotificationService`, `Board_UpdateNotificationService`
- **Example MSSQL Syntax:** `EXEC sp_xml_preparedocument docHandle OUTPUT, XmlDetail; SELECT * FROM OPENXML(docHandle, ...)`
- **Expected PostgreSQL Syntax:** `SELECT * FROM xmltable('/root/NotificationServiceDetails' PASSING XmlDetail COLUMNS ...)`
- **Recommended Fix Location:** Add XMLTABLE mapping in `BodyConverter.cs` to convert `OPENXML` and strip `sp_xml_preparedocument` calls.

#### Group 5: Identity Column on CREATE TEMP TABLE
- **Pattern:** `GENERATED BY DEFAULT AS IDENTITY` syntax error on temp tables.
- **Affected Procedures:** `Board_Authority_Select`
- **Example MSSQL Syntax:** `CREATE TABLE #T (RowNum INT IDENTITY(1,1))`
- **Expected PostgreSQL Syntax:** `CREATE TEMP TABLE T (RowNum serial)`
- **Recommended Fix Location:** In `BodyConverter.cs`, map T-SQL `IDENTITY(1,1)` on temporary tables to `serial` instead of converting to `GENERATED BY DEFAULT AS IDENTITY`.

### Contact Root Cause Groups

#### Group 1: GETDATE() Function wrapped in quotes
- **Pattern:** Date parameter default values like `'GETDATE'` instead of `CURRENT_DATE`.
- **Affected Procedures:** `Contacts_GetHistoryList`, `Contacts_GetHistoryListCount`
- **Example MSSQL Syntax:** `@SearchDate1 DATETIME = GETDATE`
- **Expected PostgreSQL Syntax:** `searchdate1 date DEFAULT CURRENT_DATE`
- **Recommended Fix Location:** Add `GETDATE` checks in `MapDefault` inside `Converter.cs` before it wraps standalone words in single quotes.

#### Group 2: SQL Comments inside Parameter blocks breaking parser
- **Pattern:** Parameters left out of signature, throwing 'variable not known' error.
- **Affected Procedures:** `Contacts_DeleteHistory`, `Contacts_SaveRestore`
- **Example MSSQL Syntax:** `CREATE PROCEDURE MyProc -- comment\n @Param INT AS`
- **Expected PostgreSQL Syntax:** `CREATE FUNCTION myproc(param integer)`
- **Recommended Fix Location:** In `Converter.cs`, strip all single-line SQL comments (`-- ...`) from `rawParams` before passing to `SplitParams`.

#### Group 3: Carriage Return lookbehind shield
- **Pattern:** Double semicolons or trailing semicolons on block keywords.
- **Affected Procedures:** `Contact_CheckInsertGroupDefault`, `Contact_InsertShareGroup`, `Contacts_DelContactsUser`, `Contacts_GetUser`, `Contacts_GetUserByShareGroup`, `Contacts_InsertGroup`, `Contacts_InsertListGroup`, `Contacts_InsertUser`, `Contacts_SaveArrange`, `Contacts_SaveArrangeLike`, `Contacts_SaveContactsHistory`, `Contacts_SetAddress`, `Contacts_SetContactsUser`, `Contacts_SetEmail`, `Contacts_SetNumber`, `Contacts_SetShare`
- **Example MSSQL Syntax:** `INSERT INTO Table ...`
- **Expected PostgreSQL Syntax:** `INSERT INTO Table ...;`
- **Recommended Fix Location:** Same as Board Group 1 (normalize `\r` to `\n` at the start of `ConvertBody`).
